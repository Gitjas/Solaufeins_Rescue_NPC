安装步骤

注意：如果你之前安装过此模组，请在提取新版本之前将其删除。为此，请卸载所有先前安装的组件，并删除模组文件夹和可执行文件。

 

安装模组时，请确保游戏未运行。

 

在安装或卸载时，请不要通过点击关闭按钮来关闭DOS窗口！相反，在被指示这样做时，请按Enter键。

关于模组化EE游戏的注意事项

对于增强版，重要的是你必须将模组安装到你正在玩的游戏的语言版本中。否则，模组的对话将不会显示，而是出现错误消息。

 

EE游戏是正在实时支持的游戏。请注意，每次补丁更新都会清除你当前的模组设置！如果你正在玩一个模组化的游戏，你可能想要延迟补丁更新，因为即使重新安装了模组，你可能也无法使用你的旧存档继续游戏。

Windows

将模组压缩包的内容解压到游戏的主目录（包含CHITIN.KEY文件的BG2/BGT/BGII:EE/EET）。解压成功后，你的游戏文件夹中应该有一个c#solaufein文件夹和一个setup-c#solaufein.exe文件。安装时，只需双击setup-c#solaufein.exe并按照屏幕上的指示操作。

 

在游戏文件夹中运行setup-c#solaufein.exe来重新安装、卸载或以其他方式更改组件。

Mac OS X

解压模组压缩包，然后将文件夹内容复制到你的游戏文件夹（包含CHITIN.KEY文件的BG2/BGT/BGII:EE/EET）。如果正确解压，你的游戏文件夹中应该有一个“c#solaufein”文件夹、setup-c#solaufein和setup-c#solaufein.command。安装时，只需双击setup-c#solaufein.command并按照屏幕上的指示操作。

 

在游戏文件夹中运行setup-c#solaufein.command来重新安装、卸载或以其他方式更改组件。

Linux

将模组内容解压到你想要修改的游戏文件夹。从https://github.com/WeiDUorg/weidu/releases下载最新版本的Linux WeiDU，并将其和WeInstall复制到/usr/bin。之后，打开终端并cd到你的BG2安装目录，运行'tolower'并回答Y两次。如果你在同一目录下已经运行过第二个选项（linux.ini），你可以避免再次运行它。如果你不确定，运行tolower并选择两个选项是安全的做法。

 

在游戏文件夹中运行WeInstall c#solaufein来安装模组。然后运行wine BGMain.exe并开始游戏。

通用信息

此模组与Imoen 4 Ever兼容。

 

对于BGII/BGT，你需要安装“Infinity Animations”的主要组件（“核心模组”），才能在游戏中看到蛛化精灵动画。如果在安装索劳纷模组的主要组件之前没有安装IA，那么可以稍后安装这个模组的组件“在此模组中启用蛛化精灵动画”，以启用蛛化精灵动画。

 

这个模组中包含一些跨模组内容。拯救索劳纷 - jastey的BGII索劳纷 NPC的主要组件应该在以下模组之后安装：

-Lava的伊莉丝翠之歌，以确保这两个模组完全兼容。

组件“为卓尔玩家角色安装检测”应该在以下模组之后安装 - 如果你的玩家角色是卓尔，并且你计划安装其中一个亚种族模组：

-Corsymyr的BG:EE亚种族模组

与Weimer的索劳纷的兼容性

如果安装了两个索劳纷 NPC模组，游戏中的处理方式如下。我的想法是让玩家能够完成救赎任务，但如果他们更喜欢Weimer的刻画，可以让Weimer的索劳纷稍后加入，尽可能减少对沉浸感的破坏。

 

这个模组不会以任何方式改变Weimer的索劳纷模组的内容。

 

Weimer的索劳纷是在从乌斯特·拿萨城出来后，在神殿仪式中背叛大祭司并拯救龙蛋后，第一个接近玩家角色的。如果你安装了Weimer的索劳纷并想玩救赎任务，你需要将Weimer的索劳纷送到地表等你。任务“拯救索劳纷”（即整个模组）只有在Weimer的索劳纷没有在幽暗地域加入团队，也没有留在乌斯特·拿萨城前面等待时才会开始。

 

在索劳纷从蛛化精灵茧中被救出后，你可以通过点击适当的回复选项，使我的索劳纷 NPC消失（即只留下Weimer的索劳纷稍后加入）。Weimer的索劳纷模组将被检测到，索劳纷将告诉玩家：“我们已经在地表上约定见面。我现在就去那里，如果你仍然希望我加入你。”

-如果你告诉他“是的，索劳纷。我会像我们在乌斯特·拿萨城城门前约定的那样，在那里等你！”，那么只有Weimer的索劳纷会在精灵营地外等你。

-如果你回答“不，这次经历改变了你。我更喜欢现在这个样子的你。”，那么我的索劳纷也将作为可加入的NPC使用。注意：这个模组不会改变Weimer的索劳纷的存在。

我的索劳纷会出现在精灵营地外的[2477.2023]坐标处（东南角）――位置和Weimer版的索劳纷在相反的一侧，这样如果你把“两个”模组的索劳纷s都送到地表，就不会同时看到两者。

 

为了让组件“在乌斯特·拿萨城给索劳纷模组的肖像”工作，它必须在Weimer的索劳纷之后安装。

详细描述

基本信息

通过此次更新，游戏内的角色索劳纷，一位来自乌斯特·拿萨城的卓尔战士，已被扩展为一个可招募的NPC。在招募之前，玩家必须将索劳纷从蛛化精灵的房间中解救出来。这个起始任务可以独立于是否招募他进行体验。

关于Weimers 索劳纷的兼容性问题已在前文详尽阐述。

 

即使队伍人数已满六人，索劳纷仍可作为“编外队员”随队行动。更多细节请参阅下文。

 

第二个可选模组组件允许玩家在游戏中为索劳纷在乌斯特·拿萨城使用模组肖像。若希望在游戏中看到这一变化，必须在安装Weimer的索劳纷组件之后安装此组件。

 

第三个可选组件适用于BGII和BGT：若已安装Infinity Animations的主要组件（详见下方“链接”部分），则该组件将被安装并应用mod中的蛛化精灵动画。若未安装此组件，mod中的所有蛛化精灵将显示为卓尔。除了外观上的差异外，游戏体验不会受到影响。 :-) 对于BGII:EE和EET，动画将被包含并自动应用。

若您计划在安装Infinity Animations Mod之前安装索劳纷的主要组件，请稍后再安装该组件。

 

第四个可选组件允许安装索劳纷的替代肖像，同样由Holger Kirste创作。

 

第五个可选组件用于检测希望在游戏中的玩家角色种族为卓尔的玩家。安装此组件后，游戏中的一个变量将被设置，以便索劳纷将玩家角色识别为卓尔。其工作原理如下：

如果未检测到亚种族mod，该组件将为精灵玩家角色在与艾达隆对话时添加一个额外的回复选项，玩家可以选择告知玩家角色是卓尔。这将在艾达隆透漏她的卓尔幻象计划后显示。如果玩家不打算以卓尔玩家角色进行游戏，这可能会破坏沉浸感。选择这个回复选项，以便索劳纷在后续见面时能识别玩家角色是卓尔。

如果检测到亚种族mod，将不会向艾达隆添加回复选项。游戏中的脚本将自动检测玩家角色是否为卓尔。考虑的亚种族mod包括：

- Corsymyr的BG:EE亚种族mod

- FinnJO的亚种族mod

- tipun的亚种族mod

- Welverin的亚种族mod

在任何亚种族mod之后安装这个组件，以确保卓尔检测功能生效。

请注意

mod需要在首次访问乌斯特·拿萨城之前安装，因为主乌斯特·拿萨城区域进行了区域修改。对于BGII/BGT和希望使用蛛化精灵动画，你可能需要开始一个新游戏。

 

索劳纷不会加入声望低于10的邪恶玩家角色队伍。这是因为我认为友谊路径最适合一个善良阵营的玩家角色。此外，在我看来，这也是合理的，因为索劳纷试图离开一个邪恶社会，不会足够信任一个声望低的邪恶玩家角色，以至于愿意将生命托付给他。

详情

mod的起点是队伍在恶魔仪式后离开乌斯特·拿萨城。显然，索劳纷必须这在之前存活。请注意与Weimer的索劳纷的兼容性，因为如果Weimer的索劳纷已经加入了队伍或在乌斯特·拿萨城等待，任务将不会开始（更多兼容性细节见下文）。

 

索劳纷将在艾达隆的巢穴入口处等待（再次提醒：只有当Weimer的索劳纷不在附近时他才会出现）。他会与玩家角色交谈，但随后会被其他卓尔捕获并带到蛛化精灵室。接下来就看玩家角色是否能救出索劳纷，避免他变成蛛化精灵了！

 

在从蛛化精灵坑中被释放后，索劳纷想收集一些东西再加入队伍。他将在队伍离开乌斯特·拿萨城前不久重新加入。此时，也可以告诉他自己出去在外面等待。然后他将在精灵营地外的坐标[2477.2023]（东南角）处等待。

 

如果玩家在ToB中开始新游戏，可以通过选择正确的回复选项来选择友谊的状态。注意：如果回答说索劳纷和玩家角色在一起冒险过，只会显示正常的ToB内容。如果回答说索劳纷在从蛛化精灵茧中被释放后不久就离开了，那么整个友谊轨迹将从头开始。后者选项是为那些对这个mod的刻画感到好奇但不想玩完整个游戏的玩家准备的。对于ToB，假设总是玩家角色从蛛化精灵室中释放了索劳纷。

内容

任务“拯救索劳纷”：在主要的幽暗地域区域添加了蛛化精灵房间（入口位于西洞壁，北边是队伍首次进入幽暗地域的地方（在一群夺心魔后面））。

 

索劳纷作为正式的队伍成员：友谊路径（有26个基础对话），NPC玩笑（与每个BioWare和BeamDog的NPC在SoA中至少有2个，在ToB中至少有1个玩笑），额外的场景对话和插话，PID（玩家发起的对话）有几个完整的对话。没有恋爱情节。

 

在阿斯卡特拉有一个小遭遇，有两个后续事件，最后一个在安姆克萨蓝（将其称为任务未免过于夸张）。玩家角色可以和平解决事件。

 

如果安装了可选组件，索劳纷会承认你的玩家角色是卓尔――但不要期望他能教授卓尔的知识，因为他试图将这一切抛诸脑后。激活需要安装第五个组件。它会检测是否在安装索劳纷模组的第五个组件之前安装了任何亚种族模组（目前只考虑英文版本，因为这些模组尚未被traify_d）：Welverin的亚种族模组，FinnJO的亚种族模组，Corsymyr的BG:EE亚种族模组，tipun的亚种族模组）。如果没有为“手工制作”的卓尔玩家角色安装任何亚种族模组，第五个组件会在艾达隆透露她的卓尔幻象计划后，为精灵玩家角色的对话添加一个回复选项，告诉艾达隆Player1是一个卓尔。只有当索劳纷应该将你的玩家角色视为卓尔时才点击这个。如果自动检测到亚种族模组，这个回复选项将不会被添加。

编外成员模式

如果你需要队伍位置给其他人，索劳纷可以被踢出队伍并被告知“跟在其他人后面”，然后他将像“雇佣兵”一样跟随，或者我称之为“编外成员”。在编外成员模式下，他需要时不时地加入队伍以升级。在编外成员模式下，索劳纷可以被选择并给予命令，以及告诉他使用快速物品栏物品和特殊能力。他所有的对话、玩笑和插话都将正常触发。注意：在编外成员模式下跟随时，玩笑的频率不会受到玩笑加速器模组的影响，因为玩笑必须由索劳纷的脚本触发，而不是引擎。不过，有三个不同的间隔计时器可用。通过PID可以改变玩笑之间的间隔时间。只需告诉索劳纷他应该更频繁或更少频繁地开玩笑，玩笑之间的计时器就会相应地改变。

 

要重新加入队伍，与他交谈并告诉他你需要“查看（他的）物品栏”，他将以全队伍成员的身份重新加入。如果你时机把握得当，在其他被踢出的NPC达到玩家角色并进行踢出后的对话之前，索劳纷在队伍中的那一刻足以让他升级并重新整理他的物品，而无需最终踢出任何其他队伍NPC来实现这一点。（除非那些NPC在没有后续对话的情况下离开。）

 

作为编外成员跟随时，他不会死亡，但会保持在1点HP。如果索劳纷在战斗中的HP降到1点，他会晕倒。治疗他不会改变这一点。他的阵营将改变为“中立”，这样敌人就会停止攻击他。一旦战斗结束，他将恢复意识，完全治愈，并重新加入队伍作为队员/雇佣兵。完全治愈看着像作弊，但确实能解决索劳芬生命值过低时归队就暴毙这个系统机制（别管它叫bug）的问题。

 

如果在战斗中及时治疗索劳芬，可以防止其生命值降至1点而倒地。若提前设置预警，他本应在生命值过低前发出警告，但该功能在BGT版本中存在异常

 

注意：作为雇佣兵，索劳纷将始终位于满编队伍最后一个队伍成员的后面，无论是行走还是切换地图。这对需要他当主力战士甚至主坦（本该顶在遭遇战第一线）的情况来说相当蛋疼。

 

注意：不幸的是，游戏中的“编外成员模式”需要特殊处理以适应所有限制：（这些提示是现有引擎限制和我所使用的解决方案的混合）。

- 在编外成员模式下，NPC周围没有“战争迷雾清除”。因此，他可能会在地图的“黑色”区域“消失”。不幸的是，这可能导致索劳纷不再被包含在“召集所有队伍成员”按钮中，尽管他仍然是绿色圆圈标记，并且一旦被一个完整队伍成员再次看到，他将正确地与队伍一起返回。如果他因为AI想要他走另一条路径到达目的地而失去对满编队伍成员的视线，这将特别烦人，需要在地图上走动直到他再次被看到。如果跟随多个“编外成员”，并且使用旧的游戏引擎，这似乎尤其成问题。

- NPC的物品栏中的物品*不*被识别为队伍拥有的物品。这是一个引擎问题。因此，如果游戏中的任务物品需要被任务给予者识别，它们就不应该在NPC的物品栏中。例如：爱文海的书，为泰龙准备的掘地虫壳等。

当切换至编外成员模式时，被标记为“关键”的任务物品会自动转移到玩家的物品栏中。这可能会导致玩家物品栏中的物品不慎掉落至地面，这属于引擎层面的问题。因此，在将NPC角色切换至编外成员模式后，务必检查是否有物品遗留在地面上。

在编外成员模式下，我的NPC在执行特定任务时获得的新物品，本应直接出现在NPC的物品栏中，却总是出现在队伍或玩家的物品栏里，这同样是引擎的问题。尽管我已编写脚本确保物品在事件发生后能正确地出现在NPC的物品栏中，它们仍然通过玩家的物品栏进行处理，导致物品可能不慎掉落至地面。

我尝试确保我的NPC的任务物品（包括游戏中的原始物品，只要它们对我的NPC任务是必需的）能够被识别并出现在NPC的物品栏中。这在以下条件下是可行的：

-- 所有相关物品在给予或从任务给予者那里接收之前，都必须通过玩家的物品栏进行处理。这可能导致其他物品意外地掉落到地面。

-- 如果物品存放在次元装备中（包括各种类型的袋子、宝石袋、药剂匣等），那么在与任务给予者的对话过程中，物品不会被移除。之后，物品仍然保留在次元装备中，如果任务脚本没有考虑到这一点（并假定物品应该消失），可能会引发问题。

-- 关于次元装备中的物品问题：如果物品存放在次元装备中，当NPC随后作为完整成员加入队伍时，游戏可能会因未知原因崩溃。这种“崩溃错误”可以通过保存游戏并重新加载来解决。

 

游戏时需要记住的事项简要总结：

在每次任务事件结束后以及切换至编外成员模式时，务必检查一下玩家是否遗失了存放在物品栏中的物品，这些物品是否需要从地面上捡回。

在将编外成员重新纳入队伍之前，请保存游戏。若游戏发生崩溃，可重新加载之前的存档以避免崩溃。

-有必要也有推荐将非关键任务物品放在正常队伍成员的物品清单中，而不是放在编外成员的物品清单中。

在接近任务发放者之前，应从编外成员的物品栏中取出次元装备中的任务物品，或者至少保存游戏以防在将NPC作为完整队伍成员之前发生“崩溃错误”。

在某些游戏过场动画中，作为编外成员，索劳纷可能会出现在他本不该出现的位置。这只能通过在相关脚本中特别考虑索劳纷来预防。一些问题将在后续版本中得到解决。

索劳纷物品栏中的物品可能不会被任务发放者识别。唯一的例外是一些我专门为它们编写脚本的自定义模组物品，这些物品会被注意到。

索劳纷没有清楚的战争迷雾能力。（原因是引擎无法处理超过8个带有CFoW的生物，包括6个队伍成员和场景探子。）因此，当作为编外成员跟随时，他可能会“消失”在视线之外并停止移动。他将不再前往会合点，而需要回到队伍成员的视觉范围内才能重新激活。

索劳纷的角色

索劳纷是一个数百岁的卓尔（尽管还不算“老”），他在精神上与卓尔文化分离。他被困在残酷的卓尔社会中，他认为是一个僵局，于是通过自己获得一些地位成为男性指挥官，并得到一个强大家族的支持（家族的名字从未提及，也不重要）。这样他得到了一定的保护，他以讽刺的方式发泄他的不满，并以一种不典型的卓尔方式反对较弱的女性。他对菲里的爱早已消失，他像鄙视所有寻求蜘蛛女王罗丝青睐的其他卓尔一样鄙视她。

 

模组对话的主要主题是离开地底世界给他带来的变化，特别是失去大部分魔法和被阳光致盲导致战斗能力减弱。同时，他从卓尔的角度评论了地表居民向往的生活方式。

 

他会与愿意与他交谈并以良好或中立方式处理任务的玩家角色成为朋友。对他，卓尔，持怀疑态度是一种明智的情感。他几乎不会因为任何玩家角色的评论而感到被冒犯。

 

注意：友谊路径也处理了索劳纷实际允许友谊感觉的问题，有时可能有些刻板。没错，可能有些陈词滥调。

已知问题

索劳纷的魔法抗性变得过高：我得到的EEs值高达127%，而BG2/BGT的值为88%，而不是最大值80%。我不知道如何修复这个问题。这个值也高于cre文件中设定的值，所以似乎有些我不了解的游戏内影响因素。

在编外成员模式下游戏时需要记住的简短摘要：

-每次任务事件后以及切换至编外成员后，检查玩家角色是否丢失了应从地面拾取的物品。

-在将编外成员重新纳入队伍之前保存游戏。如果游戏随后崩溃，重新加载存档。这应该可以防止崩溃的发生。

-有必要且建议的是，非关键的任务物品不应存放在编外成员的背包中，而应存放在普通玩家的背包里。

-在接近适当的任务发放者之前，从编外成员的物品栏中取出次元装备中的任务物品，或者至少保存游戏以防止“崩溃漏洞”。

在某些游戏场景中，作为编外成员，索劳纷可能会出现在他不应该出现的位置。我不确定如何解决这个问题。

-索劳纷物品栏中的物品不会被任务发放者识别。例外是一些我特别编写脚本的自定义模组物品，它们会被注意到。

索劳纷没有清楚的战争迷雾能力。（原因是引擎无法处理超过8个带有CFoW的生物，包括6个队伍成员和场景探子。）因此，当作为编外成员跟随时，他可能会“消失”在视线之外并停止移动。他将不再前往会合点，而需要回到队伍成员的视觉范围内才能重新激活。
